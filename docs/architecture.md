# Architecture — v2.4.1

The application separates five responsibilities:

| Layer | Primary files | Responsibility |
|---|---|---|
| Local source import | `app/js/elmy-importer.js`, `app/vendor/pdfjs/`, diagnostic difficulty maps | Offline PDF text extraction, Elmy format parsing, numbering conversion, source validation, and individual-case construction |
| Template specification | `config/brand-config.json`, `app/js/roadmap.js`, `original-reference/` | A4 design, assets, brand content, page structure, editorial rules |
| Planning engine | `config/lesson-catalog.json`, `config/calculation-rules.json`, diagnostic templates, `app/js/engine.js` | Calibrated rubric, accuracy, coverage, hours, budget allocation, homework, sessions, group logic |
| Case inputs | input schemas and imported/entered cases | Supplied student or group information only |
| Calculated results | `engine.js` output and result schemas | Question and lesson analysis, totals, homework, schedule, audit, validation |
| Roadmap view model | `app/js/viewmodel.js` | Concise student-facing data consumed by the seven-page renderer |

Data flow:

```text
Elmy PDF → local PDF.js text extraction → registered Elmy parser
                                        → validated individual case

case input → analyzeIndividual / analyzeGroup → calculated results
                                              ├→ instructor report
                                              └→ roadmap view model → seven-page roadmap
```

The PDF layer never calls the planning engine while extracting. It creates the same schema-compatible individual case used by manual, JSON, and CSV entry. `app.js` runs importer validation and a normal engine validation pass before changing application state. A small parser registry in `elmy-importer.js` isolates the currently supported `Diagnostic Exam 2026` layout and provides the extension point for future deterministic report formats.

## Configuration source of truth

The JSON files under `config/` are canonical. This includes `config/diagnostic-difficulty-maps/diagnostic-exam-2026.json`, which the Elmy importer checks for all 98 questions and for the 22/35/41 difficulty checksum. `app/js/config-data.js` is an offline mirror generated by `optional-tools/build-config-js.py` because local browsers commonly block JSON `fetch()` under `file://`.

## PDF runtime and security boundary

PDF.js 4.10.38 is bundled under `app/vendor/pdfjs/`; there is no CDN or runtime download. The UI supplies a local worker URL and the bundled worker also supports PDF.js's same-file fallback when a browser restricts `file://` workers. Import is capped at 25 MB, disables PDF.js evaluation, reads only the text layer, and never executes PDF attachments or embedded scripts. Extracted text is assigned through `textContent` in the import UI or passed through the existing HTML-escaping report renderers.

The lesson catalog stores both explanatory rubric scores and the authoritative `calibrated_base_hours`. The engine does not infer the base time from a universal sum conversion.

## Time-allocation invariant

Lesson values are retained without lesson-level rounding. The engine applies all program-floor additions and maximum-budget reductions to named lesson or strategy components, then recomputes the actual component sum.

Individual invariant:

```text
unrounded component total = English live hours
                          + Math live hours
                          + live strategy components

14 ≤ unrounded component total ≤ 35

final scheduled total = quarter-hour rounding(unrounded component total)
```

Group invariant:

```text
unrounded core total = core English lessons
                     + core Math lessons
                     + group orientation/review/strategy

configured minimum ≤ unrounded core total ≤ configured maximum
```

Supplemental subgroup or individual support is stored separately from the capped common core.

## Rendering

Reports open in new tabs through `document.write`; a `<base>` tag points to the application directory so local assets resolve. Print CSS uses A4 pages. The roadmap renderer contains no calculation logic and receives only its view model.
